EGMOR / EGM — 完整工作逻辑与可配置选项

基于仓库当前 Python 源码整理：文本 → 360° ERP → DAP 深度与解析法线 → 透视切分 → COLMAP →（可选 4×）→ Gaussian Splatting。表格内为各阶段可对比选项。

1. 项目在做什么

目标：用自然语言描述静态空室内场景，生成可漫游的 3D Gaussian Splatting（.ply）。

核心链路：LLM 扩写场景 → 9 个位姿各一张 ERP RGB → DAP-V2 深度 + 解析法线 → 透视图 + COLMAP + 初始点云 → 可选 Real-ESRGAN 4× 与重建透视 → splatfacto 或 FastGS 训练 → 浏览器 viewer。

FAQ：Pose、首张图、Warp、OpenAI 官方 gpt-image-2

Pose 是否从第一张全景估计？

否。 九个 xyz 与 R 由 cuboid + poses.* 在 poses.py 中解析生成，在调用图像 API 之前已固定（见 poses.json），不是从首张图做视觉定位或 COLMAP BA 反推。

那「后面位置」如何谈准确？

区分两类问题：（1）外参几何——由你配置的虚拟房间决定，用于 warp / 透视采样 / 写 COLMAP；（2）多视角 RGB 是否像同一间房——靠提示词与 warp_inpaint（中心 DAP 深度 + forward_warp_erp + mask inpaint）、i2i 或裸 generate。深度每张单独 DAP，是相对深度经分位数映射到 [depth_near_m, depth_far_m] 的工程标定，不是激光雷达真值。

Warp 的作用

warp.py：源 ERP 像素 × 源深度 → 世界点 → 投到目标 ERP，z-buffer 合成；空洞膨胀后作为 OpenAI images.edit 需编辑区域，使角点图与中心诱导的 3D 结构对齐。

OpenAI 官方、效果、速度

官方：provider=openai 且默认主机或透传 images 的中继。warp_inpaint 依赖标准 images.edit。效果与速度随模型、分辨率、quality 与网络变化；DAP 一般为 GPU 亚秒～数秒级。

官方 gpt-image-2 一次出图（docs/）

运行 python scripts/docs_sample_gpt_image2_official.py（需环境变量 OPENAI_API_KEY）生成 docs/openai_official_gpt_image2_sample.png 与元数据 docs/openai_official_gpt_image2_sample.json。无密钥时仅写入 ok:false 的 JSON，不生成 PNG。

$env:OPENAI_API_KEY = "sk-..."
python scripts/docs_sample_gpt_image2_official.py

2. 端到端流水线

用户场景文本
    → SceneExpander（可 --no-expand 跳过）
    → build_pose_set（9 Pose）
    → run_hybrid_nvs（LLM RGB + DAP + 法线）
    → split_all_to_perspectives
    → build_init_pcd + write_colmap_sparse
    →（可选）sr_erp_4x.py / regenerate_persp_4x.py
    → train_gs.py → serve_viewer.py

主入口：scripts/generate_erp.py 中 generate() 完成 Stage 1。

3. Stage 1 模块逻辑

3.1 配置

erpgen/config.py：load_config + OmegaConf dotlist；resolve_run_dir 写入 outputs/runs/<run_id>/ 与 resolved_config.yaml。

3.2 位姿 poses.py

世界系 Z 向上；相机 R 为 world_from_camera，视线 +X。长方体由 cuboid.* 定义；corner_lookat：center | outward | random | level（角点与中心同向，极点对齐）。poses.generation：auto_8_corners 或显式列表。

3.3 场景扩写 scene_expander.py

Chat 补全返回 JSON → SceneSpec；prompts.build_prompt 生成每位姿 ERP 提示。

3.4 NVS nvs.py

中心：generate_rgb。角点：由 nvs_strategy.corner_method 决定。之后固定 estimate_erp_depth(..., mode="direct") 与 normals_from_erp_depth。若 warp/i2i 失败则后续角点退化为纯 generate。

corner_method行为provider 注意
warp_inpaint中心 RGBD 前向 warp → 空洞 mask → images.edit 只填洞需 openai 标准路径
i2i整图参考 edit / chat i2iopenai 或 openrouter
generate无参考文生图任意

3.5 图像 API openai_erp.py

openai：images.generate / images.edit（含 mask），可配 base_url 走兼容中继。openrouter：仅 chat 多模态，输出约 1024²，无真 mask edit。请求 SHA256 磁盘缓存。

3.6 DAP dap.py

nvs.dap_model_size：small | base | large。库内支持 cubemap_split，但 NVS 当前写死 direct。

3.7 透视切分 erp_to_persp.py

当前 _faces_for_scheme 仅支持：cubemap（6）、persp48_zigzag（48，列优先 zigzag）。docstring 中提到的 persp16 若传入会 ValueError。

perspective.fov_deg、out_size 控制视场与分辨率。

3.8 COLMAP

init_pcd 使用 fastgs.init_max_points、init_voxel_m；write_colmap_sparse 受 fastgs.init_from_points3d 控制是否写入 points3D。

3.9 Decoder 实验

decoder_experiment.enabled 时，对指定 pose 调用 LLM 生成深度/法线图，与 DAP 对比写入 decoder_vs_dap.json；主几何仍用 DAP。

4. Stage 1.5：sr_erp_4x.py

RGB：Real-ESRGAN + 水平 wrap padding。深度/法线：双线性 4×，不重跑 DAP。

5. Stage 2：train_gs.py

splatfacto：nerfstudio；失败可自动 fallback FastGS。fastgs：子进程，缺失时可 fallback 点云 PLY。
