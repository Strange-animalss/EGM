# 环境变量与 API 密钥配置说明（EGMOR）

**请勿把真实 `sk-...` 密钥写进 Git 仓库、不要发到聊天、不要提交 `config/default.yaml`。**  
仓库已把 `.env` 列入 `.gitignore`；密钥应只存在于 **本机用户环境** 或 **本机私有文件**（且不备份到公开网盘）。

---

## 1. 本仓库「从哪里读密钥」


| 用途                                    | 配置位置                                                                  | 代码行为                                                                                                 |
| ------------------------------------- | --------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------- |
| 图像 API（`gpt-image-2` 等）               | `config/default.yaml` → `openai.api_key_env`（默认 `**OPENAI_API_KEY`**） | `erpgen/openai_erp.py`：`cfg.api_key` 为空则从 `**os.environ[api_key_env]`** 读取                           |
| 文本扩写（SceneExpander）                   | 同上 + `text_model_api_key_env`（默认也是 `**OPENAI_API_KEY`**）              | `erpgen/scene_expander.py`：同样从环境变量读                                                                  |
| **本机 YAML 覆盖（推荐替代「写死 default.yaml」）** | `**config/secrets.local.yaml`**（已 `.gitignore`，需自建）                   | `erpgen/config.py` 的 `load_config`：**在** `default.yaml` **之后合并**该文件，可写 `openai.api_key: "..."` 仅留在本机 |


也就是说：**优先用环境变量 `OPENAI_API_KEY`**；若坚持要放在 YAML 里，请用 `**config/secrets.local.yaml**`（从 `secrets.local.yaml.example` 复制），**不要**把密钥写进已入库的 `default.yaml`。

## 2. Windows 上「环境变量」配在哪里

### 2.1 图形界面（对用户永久生效，新开终端也有效）

1. **Win + R** → 输入 `sysdm.cpl` → 回车
2. **高级** 选项卡 → **环境变量(N)...**
3. 在 **「用户变量」**（仅当前登录用户）或 **「系统变量」**（整机）中 **新建**：
  - 变量名：`OPENAI_API_KEY`  
  - 变量值：你的密钥
4. 确定保存后，**重新打开** Cursor / PowerShell / CMD，再跑 `python scripts/...`。

### 2.2 仅当前 PowerShell 窗口（临时）

```powershell
$env:OPENAI_API_KEY = "你的密钥"
python scripts\generate_erp.py --scene "..."
```

关闭窗口后即失效。

### 2.3 用户级持久（PowerShell，写入注册表，慎用）

```powershell
[System.Environment]::SetEnvironmentVariable("OPENAI_API_KEY", "你的密钥", "User")
```

新开终端生效。密钥会留在本机用户配置中，仍不要共享屏幕/录屏泄露。

## 3. Cursor / VS Code 里相关说明

- **集成终端**默认继承 **Windows 用户环境变量**（你在 2.1 里配好后，Cursor 内新开终端一般可直接用）。  
- **不建议**把密钥写进工作区 `.vscode/settings.json` 的 `terminal.integrated.env.windows`（容易随仓库同步或进备份）。若个人单机且仓库永不外发，技术上可行，但风险自负。  
- **Agent / 后台任务**使用的环境：以 **Cursor 启动进程时的环境** 为准；若 Agent 里读不到密钥，多半是 **在系统里刚加完变量但未重启 Cursor**。

---

## 4. 可选：仓库根目录 `.env`（本仓库脚本多数不自动加载）

- 根目录可自建 `**.env`**（已被 `.gitignore` 忽略），内容为一行：  
`OPENAI_API_KEY=你的密钥`  
- **注意**：当前主流程脚本（如 `generate_erp.py`）**不会自动读取 `.env`**，除非你自行安装并使用 `python-dotenv` 或在入口显式加载。  
- 更简单的做法仍是：**用 Windows「环境变量」或终端里 `$env:...`**。

测试脚本支持从文件读首行（不落库、需自行创建文件）：

```powershell
python scripts\quick_test_report_and_erp.py --key-file C:\Users\<你>\.openai_key.txt
```

**不要**把 `sk-...` 写进会被 Git 跟踪的文件。

---

## 5. 与 `config/default.yaml` 的关系

- `openai.base_url`：指向 **官方**（留空 / 默认）或 **兼容中继**（如你当前的 `api.chatfire.cn`）。  
- 密钥仍优先走 `**OPENAI_API_KEY`**（或 `api_key_env` 指定的名字），与 `base_url` 无关。

---

## 6. 密钥泄露后怎么办

若密钥曾出现在聊天、截图或公开仓库：**立刻在 OpenAI 平台删除该 key 并新建**；旧 key 作废后无法恢复。
